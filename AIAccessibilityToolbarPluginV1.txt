A Material Design accessibility toolbar with an AI-powered "summarize this page" button backed by the Gemini API — securely called from your own server.

== Description ==

**AI Accessibility Toolbar & API** adds a floating accessibility widget to the front-end of your WordPress site. Visitors can:

* 📝 **Summarize the current page** using Google's Gemini API (call is proxied through your server — your API key is never exposed).
* 🔤 Toggle a dyslexia-friendly font (OpenDyslexic / Lexend / Comic Sans MS fallback).
* 🌗 Enable a high-contrast dark mode.
* 📏 Toggle a reading ruler that follows the cursor.
* 🔗 Highlight every link on the page.
* 🚫 Hide distracting images, videos, and iframes.

= Security & privacy =

* Your Gemini API key is stored server-side in the WordPress options table and never sent to the browser.
* The REST endpoint that talks to Gemini is protected by a WordPress nonce and an IP-based rate limiter (5 requests per minute by default).
* All output is escaped through the WordPress escaping API, and settings-form submissions are protected by `wp_nonce_field()` + `wp_verify_nonce()`.
* The API key is sent to Google via the `x-goog-api-key` request header (not the URL), so it does not appear in access logs.

= Requirements =

* WordPress 5.8 or newer
* PHP 7.4 or newer
* A Gemini API key (get one at https://aistudio.google.com/)

== Installation ==

1. Upload the plugin folder to `/wp-content/plugins/` **or** install it via the *Plugins → Add New* screen.
2. Activate the plugin through the *Plugins* menu in WordPress.
3. Go to **Settings → AI Toolbar** and paste your Gemini API key.
4. Visit any front-end page. A ✨ button will appear in the bottom-right corner.

== Frequently Asked Questions ==

= Where do I get a Gemini API key? =

Sign in to [Google AI Studio](https://aistudio.google.com/) and create a key. Free-tier limits apply.

= Which Gemini model does the plugin call? =

`gemini-3.5-flash` via the `generateContent` endpoint. You can change the model by editing the `AI_TOOLBAR_MODEL` constant in the main plugin file.

= Is the widget accessible? =

The trigger button and dialog panel expose `aria-label`, `aria-expanded`, `aria-controls`, and `role="dialog"` attributes. The toast uses `role="status"` with `aria-live="polite"`. The panel closes when the visitor presses **Escape**.

= Does it work on sites without pretty permalinks? =

Yes — the REST endpoint URL is generated with `rest_url()`, so it works with any permalink configuration and with subfolder installations.

= What happens if I hit the rate limit? =

The server returns HTTP 429 and the widget shows a toast asking the visitor to try again shortly. The default limit is 5 requests per minute per IP address; change `AI_TOOLBAR_RATE_LIMIT` / `AI_TOOLBAR_RATE_WINDOW` in the main plugin file to adjust.

= Will uninstalling the plugin delete my API key? =

Yes. When you delete the plugin through the WordPress admin, `uninstall.php` removes the stored API key and any lingering rate-limit transients (per-site on multisite).

== Privacy ==

When a visitor clicks **Summarize Page Content**, the following data is sent to Google's Gemini API from your server:

* The visible text of the current page (truncated to 12,000 characters).
* Your Gemini API key (in the request header).

No visitor information (IP address, cookies, user agent) is forwarded to Google — the request originates from your web server. IP addresses are, however, hashed and stored temporarily (up to 60 seconds) in the WordPress transients table for rate-limiting purposes.

Review Google's [Generative AI terms](https://ai.google.dev/terms) before enabling this feature on production sites, and disclose the use of AI processing in your site's privacy policy where required.
